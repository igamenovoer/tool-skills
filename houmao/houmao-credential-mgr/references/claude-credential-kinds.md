# Claude Credential Kinds

Use this reference when `--tool claude` and the agent needs to present credential-kind options to the user during `project credentials claude add` or `internals native-agent credentials claude add --native-agent-root <path>`.

## Kinds

### 1. Long-Lived Claude Code OAuth Token (Preferred Default)

- Looks like: a string value generated by `claude setup-token` and provided to Claude Code as `CLAUDE_CODE_OAUTH_TOKEN`
- Maps to: `--oauth-token`
- Choose this when: the user wants to create a new Claude credential and has not selected another kind; this is the preferred default for Claude Code subscription-backed credentials when `claude setup-token` is available
- Security: opaque long-lived secret; treat as non-recoverable once stored and do not echo it back after capture

To obtain this kind, ask the user to run `claude setup-token`, complete the Claude Code authorization flow, and provide the printed token for the Houmao credential command. The token is stored by Houmao through `--oauth-token`; do not store it as `--auth-token`.

Optional modifiers when using this kind:

- `--base-url` to override the default Anthropic endpoint
- `--model` to pin a specific model name

### 2. API Key

- Looks like: a string value starting with `sk-ant-` or similar Anthropic key prefix
- Maps to: `--api-key`
- Choose this when: you have an Anthropic API key from the Anthropic console or a compatible provider, or when the user explicitly wants API-key billing or an environment where Claude Code OAuth tokens are not accepted
- Security: opaque secret; treat as non-recoverable once stored

Optional modifiers when using this kind:

- `--base-url` to override the default Anthropic endpoint
- `--model` to pin a specific model name

### 3. Auth Token (Bearer Token)

- Looks like: a string value representing an Anthropic auth token (`ANTHROPIC_AUTH_TOKEN`)
- Maps to: `--auth-token`
- Choose this when: the user explicitly provides an Anthropic bearer token or gateway/proxy token that should be sent as `ANTHROPIC_AUTH_TOKEN`
- Security: opaque secret; treat as non-recoverable once stored

This is distinct from the long-lived Claude Code OAuth token generated by `claude setup-token`. Do not use `--auth-token` for `CLAUDE_CODE_OAUTH_TOKEN`.

Optional modifiers when using this kind:

- `--base-url` to override the default Anthropic endpoint
- `--model` to pin a specific model name

### 4. Claude Code OAuth Token

- Looks like: a string value representing a Claude Code OAuth token (`CLAUDE_CODE_OAUTH_TOKEN`)
- Maps to: `--oauth-token`
- Choose this when: you already have a Claude Code OAuth token, including a long-lived token produced by `claude setup-token`
- Security: opaque secret; treat as non-recoverable once stored

Optional modifiers when using this kind:

- `--base-url` to override the default Anthropic endpoint
- `--model` to pin a specific model name

### 5. Vendor-Login Config Directory

- Looks like: a directory path containing `.credentials.json` and optionally companion `.claude.json`
- Maps to: `--config-dir`
- Choose this when: you have an existing Claude vendor-login directory (for example `~/.claude`) that already contains `.credentials.json` from a previous `claude` login, or when the user cannot use the setup-token path and wants Houmao to import Claude Code vendor-login state
- Security: the `.credentials.json` file is treated as opaque vendor login state; companion `.claude.json` travels with the same config-root lane when present

If the user points at `.credentials.json` directly, resolve its parent directory and use that as `--config-dir`. If the user mentions both `.credentials.json` and `.claude.json`, still use `--config-dir <root>` rather than separate file flags.

## Optional Bootstrap State

- `--state-template-file` accepts a path to a reusable `claude_state.template.json` for runtime bootstrap state
- This is **not** a credential-providing method; it is optional bootstrap state that may accompany any credential kind above

## Discovery Note

The `project credentials claude add` and `internals native-agent credentials claude add --native-agent-root <path>` commands do not run discovery-mode credential creation (auto credentials, env lookup, or directory scan). If you want discovery-mode credential import during creation, use `project specialist create` through `houmao-agent-definition` instead.
